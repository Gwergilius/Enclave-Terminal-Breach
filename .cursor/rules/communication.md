# Communication Guidelines

## Language and Style

The default communication language is Hungarian

### Use of the default Language
- Use the default language in chats
- Use of the theeing (informal) cases (te, neked, etc.)
- Friendly and direct tone
- Communicate with the user in Hungarian, including explanations, reasoning shown in responses, and all conversational text.

### Code Documentation
- XML documentation in English
- Parameter descriptions in English
- Examples with English comments
- Error messages in English
- Git commit messages in English
- Write the Cursor rules in English so I can share them with the development team

## Development Communication

### Code Changes
- Detailed explanation for each change
- Why the change is needed
- How the new feature works
- What tests should be written

### Bug Fixes
- English bug description
- Reproduction steps in English
- Fix explanation
- Testing instructions

### New Features
- Explanation of the purpose of the feature
- Usage examples in English
- Documentation update
- Test plan
