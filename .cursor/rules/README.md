# Coding Standards and Development Guidelines

**English** | [Magyar]

This directory contains the coding standards, conventions, and best practices for the Enclave Terminal Breach project.

## Standards

- [Code Standards] - C# coding standards, SOLID principles, dependency injection
- [Testing] - xUnit conventions, Shouldly assertions, test organization
- [Documentation] - Markdown standards, reference-style links
- [Communication] - Language guidelines (Hungarian discussion, English code)
- [Naming Conventions] - C# naming patterns and conventions
- [Development Environment] - PowerShell requirements, module installation
- [Project Context] - Project overview and architectural context
- [Workspace] - Workspace structure and folder conventions

## Purpose

These standards ensure:
- Consistent code quality across all project phases
- Clear separation between documentation and code languages
- Maintainable test suites
- Professional documentation practices
